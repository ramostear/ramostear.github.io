异常信息描述：

HTTPStatus500-/WEB-INF/jsp/worklog/editWorkLog.jsp (line: 60, column: 56) The JSP specification requires that an attribute name is preceded by whitespace

原因分析：

异常信息翻译为：JSP规范要求一个属性名称前面需要有空格
知道错误原因后，就知道如何解决了，开发的时候使用的是Tomcat6.0的服务器，当项目迁移到服务器进行部署后，服务器使用的是Tomcat7.0，JSP规范要求每个属性之间必须要以空格隔开。因此找到报错的地方，我的报错地方是：/WEB-INF/jsp/ worklog/editWorkLog.jsp(line: 60, column: 56)。在这里有一个属性没有用空格隔开，添加一个空格即可。
报错代码：

path和style两个属性没有用空格分开。

备注：以后遇到这样的错误，先检查一下相关页面的编码规范，JSP页面中的标签属性要按照jsp的规范进行书写，每个属性“键=值”后要加上空格。其实这是一个很低级的错误，养成良好的编码习惯很重要。
